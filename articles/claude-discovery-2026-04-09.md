# Claude Code 今日の発見 — 2026-04-09

「寝ている間にコードを書かせる」がもはや比喩ではなくなった。今日見つけた事例は、AIを"便利なアシスタント"から"夜勤のチームメイト"へ昇格させる動きが加速していることを示している。

## Stripeが1,370人のエンジニアにClaude Codeを導入、10週間の移行作業を4日で完了

決済インフラ大手のStripeが、全社規模でClaude Codeを展開した事例が公開されている。1,370人のエンジニアに対してゼロコンフィグで配布し、開発インフラチームを率いるScott MacVicarの指揮のもと、10,000行のScalaコードをJavaへ移行するプロジェクトを実施。従来手法では10週間のエンジニアリング工数と見積もられていた作業が、わずか4日で完了した。

興味深いのは、Stripeが「AIは自律的に動く代替要員ではなく、文脈が必要な有能な新入りエンジニア」という位置づけで社内教育を行っている点。署名付きエンタープライズバイナリをAnthropicと共同開発し、セキュリティ懸念をクリアしたことで全社導入が可能になった。大規模な技術負債を抱えるクライアントへの移行支援提案で、この事例は説得力のある前例になりそうだ。

[Stripe Customer Story](https://claude.com/customers/stripe)

## GMOが公開した「Sleepship」、寝ている間に開発を自律的に進めるGoツール

GMOインターネットグループが開発したSleepshipは、タスクファイルを定義しておくとClaude Codeがバックグラウンドで自律的に開発を進めるGoツール。名前の通り「寝ている間も開発が進む」コンセプトで、ビルドエラーやテスト失敗時には自動で最大3回リトライし、前回のエラー情報をプロンプトに含めて修正を試みる。

実装の核はexec.Commandによる子プロセス制御。大規模リファクタリングでは、AIが自動的に複数のサブタスクに分割して段階的に実行する。再帰実行機能により「調査→計画→実装」という段階的フローを最大3階層まで自動化できる。夜間バッチ的にコード整理を走らせておいて、朝出社したらPRが並んでいる、という運用が現実的になってきた。

## 広告レポート業務が3〜5時間から7分に、Rimoの実務事例

Rimoの佐伯氏が公開した事例では、BigQueryに格納された広告データから数値を抽出し、状況の要約・考察・改善提案まで出すスキルを自作。管理画面を見てキーワードや検索語句を精査し、改善案を考えるまでの作業が、従来の3〜5時間から体感7分にまで短縮された。

ポイントは「AIが出した提案をそのまま使わない」設計。前提情報が薄いと精度が落ちるため、最終判断は人間が持つ。現在は広告運用の改善観点を70〜100項目規模のチェックリストとしてスキル化し、「世の中の『広告はこうあるべき』という観点」を取り込む構想を進めているという。代理店業務のレポート作成で、AIにチェックリストを持たせて一次分析させる形は、すぐ試せそうなアプローチだ。

[Rimo](https://rimo.app/@rimo/general/claude_code_ad_reporting)

## GitHubを「Claude Codeの第二の脳」に変える完全自動化ワークフロー

GitHubで公開されたclaude-code-github-workflowは、計画からデプロイまでを自動化する8つのGitHub Actionsと8つのスラッシュコマンド、4つの自律型エージェントをパッケージ化したもの。計画JSONをGitHub Issueに変換し、自動でブランチを作成、PRを生成、品質検査を通してマージ、リリースまで一気通貫で流せる。

セットアップは5分で完了し、開発生産性が3〜5倍向上するとの触れ込み。fork安全性、レート制限、無限ループ防止といったセキュリティ機能も搭載している。Claude Codeを使い始めたが「結局git操作は手動」という状態から一歩進めたいときに、参考になる設計だろう。

[GitHub: claude-code-github-workflow](https://github.com/alirezarezvani/claude-code-github-workflow)

## Anthropicの隠しプロジェクト「Conway」、24時間稼働AIエージェントの設計思想

2026年3月末にリークされたAnthropicの内部プロジェクト「Conway」は、従来の「呼び出し待ち型」とは異なる「24時間365日稼働する自律エージェント」の設計を示している。メール受信、GitHub更新、Slack通知といった外部イベントにWebhook経由で自動反応し、人間が寝ている間も作業を継続する構造だ。

Qiitaで公開された解析記事では、15秒間隔のハートビートで定期的にタスクを確認するデーモン方式、Express.jsによるWebhook受信サーバー、セッション間で永続するメモリ構造といった実装パターンが紹介されている。毎晩3時にメモリを統合する「autoDream相当」の機構も提示されており、常駐型AIの実装を検討している向きには具体的なヒントになる。

## コンテンツパイプライン全自動化、659行のスキルファイルで実現

開発者のIgor Gridel氏は、ソーシャルメディア投稿のワークフロー全体をClaude Codeで自動化した事例を公開している。659行のスキルファイルでPost For Me APIの使用方法をClaudeに教え、3つの自動エージェントを構築。4時間ごとにObsidianの受信箱と下書きをチェックする「オペレーションスイープ」、毎朝9時に前日の投稿活動とエンゲージメントを分析する「朝のブリーフィング」、土曜朝に週間パフォーマンスを分析する「週間ストラテジスト」が動く。

自動投稿ではなく「何が重要かを提案する」システムに転換したことで、より実用的な結果が得られたという。マーケティングチームのコンテンツ運用で、投稿判断は人間に残しつつ分析と優先順位付けをAIに任せる設計は、いきなり全自動を目指すより現実的かもしれない。

[Dev.to: Content Pipeline Automation](https://dev.to/igorgridel/i-automated-my-content-pipeline-with-claude-code-heres-everything-1a58)

## Claude Code 2の自動化スタック：/loop、Scheduled Tasks、Skills 2.0

3月にリリースされたClaude Code 2の新機能が、完全な自動化スタックを形成している。`/loop`はセッション内で動く定時スケジューラで、「/loop 5m check deployment」のように自然言語で定期実行タスクを設定できる。最大50個の並行タスクが可能だが、ターミナルを閉じると消える短期監視向け。

Scheduled Tasksはデスクトップアプリに統合された永続的な自動実行機能で、PCのスリープやアプリ再起動後も動作する。「最後の7日間で見落とされた実行を1回だけ実行」という仕様が独特だ。Skills 2.0では、Anthropicが自社スキルをテストした結果「6つ中5つの文書作成スキルで品質低下を発見」したことを受け、4つの専門ロールがスキルの有効性を継続測定する仕組みが導入された。監視・定期実行・品質担保という3層が揃ったことで、Claude Codeを業務プロセスに組み込む設計の自由度が上がっている。

[Medium: Claude Code 2 Automation](https://medium.com/@ai_transfer_lab/claude-code-2-automation-loop-scheduled-tasks-and-skills-2-0-explained-58c034d68de6)
