# Claude Code 今日の発見 — 2026-04-08

## Polymarketの自動取引システムを会話一本で構築、1週間で9.6%リターン

アイスランドのエンジニア、Örvar Karlssonが予測市場Polymarketの自動取引システムをClaude Codeとの対話だけで作り上げた。Polymarketは暗号通貨ベースの予測市場で、選挙や経済イベントの結果に賭けられるプラットフォームだ。

最初はCLIラッパーから始まり、価格監視、アービトラージスキャナー、取引実行、学習ループへと5日間で進化した。特筆すべきは「NegRiskスキャナー」で、複数選択肢の市場でYES価格の合計が1ドルを超えるケース（＝確実に利益が出る裁定機会）を自動検出する。初回スキャンで182件のアービトラージ機会を発見したという。

1週間の実績は8取引で勝率75%、投資額148.60ドルに対して14.22ドルの利益、ROI 9.6%。NegRiskアービトラージは100%勝率で36.2%のリターンを記録した。「退屈な戦略が一番うまくいく」というのがKarlssonの結論だ。6モジュール、50テストの完全なシステムが、Claude Codeとの単一の会話セッションから生まれた。広告運用のアービトラージ検出にも応用できそうな発想。

## レイオフから2週間、転職活動を「1日仕事」から「1-2時間」に圧縮

2026年2月にレイオフされたソフトウェアエンジニアのRobert Finesが、転職活動全体をClaude Codeで自動化した記録。

毎朝7時に求人サイトをスクレイピングして条件に合う案件をスプレッドシートに落とし、Gmailと連携して応募状況を自動更新、毎朝8時にシステム設計の面接問題を出題し間隔反復で苦手問題を繰り返す、毎日Reactのコーディング課題を出してGitHubリポジトリに保存、夕方6時に技術トレンドのブリーフィングを生成——という5つのスケジュールタスクを構築した。

さらに、職務経歴書を求人内容に合わせて自動調整し、カバーレターを生成する「Retold」というSaaSまで作ってしまった。「以前は丸一日かかっていた作業が1〜2時間で終わる」とのこと。クライアントへの定期レポーティングや競合監視にも同じパターンが使える。

[Medium - Robert Fines](https://medium.com/@fines.robert/i-automated-my-job-search-with-claude-heres-exactly-how-5d39b67938cf)

## TVerのiOSチーム、「途中参画でも一定品質」を実現するバグ修正ワークフロー

動画配信サービスTVerのiOSエンジニアが、プロジェクト途中から参画する際の課題をClaude Code Skillsで解決した事例。

3つのスキルを順番に実行する仕組みを作った。`/analyze-and-create-issues`で不具合を分析しGitHub Issueを自動生成、`/implement-issue`でIssueに基づいて実装からビルド確認・コミットまで実行、`/create-pr`でAndroid実装との比較表やBefore/Afterを含むPR本文を生成する。

ポイントは「複数リポジトリ間の比較」ができること。iOSとAndroidの実装差異を確認しながら修正方針を立てられる。プロジェクト背景を完全に理解していないメンバーでも、一定品質のアウトプットを安定的に出せるようになった。属人的ノウハウの標準化という点で、広告運用チームの引き継ぎやオンボーディングにも参考になる。

[TVer Tech Blog](https://techblog.tver.co.jp/entry/claude-code-use)

## ヘンリー社、「実装→レビュー」の反復ループを全自動化

医療系スタートアップのヘンリー社が「Orchestrator型Skill」と呼ぶ仕組みで、Claude Codeによる開発を全自動化している。

従来の問題は「AIとのリアルタイム会話に人間が付きっきりになる」こと。解決策として、親Skillが各ステップの呼び出し順序を管理し、各ステップは独立したSubAgentとして実行される構造を採用した。実装Agentとレビュー Agentをペアで配置し、レビューで問題が見つかれば修正を依頼する反復ループが回る。

これにより「全自動でも品質低下しない」状態を実現。修正不要でマージできるPRが増えたという。人間が介在するのは最初の指示出しと最終確認だけ。広告クリエイティブの制作→チェック→修正ループにも同じ構造が使えそう。

[ヘンリー エンジニアブログ](https://dev.henry.jp/entry/claude-code-orchestrator)

## 非エンジニアCPOが作った「毎朝7時に動く11個の自動化」

アソビュー株式会社のCPO・横峯樹氏が、派手さはないが毎日使い続けている11個の自動化ジョブについて語っている。

「Claude Codeで簡単なアプリを作った」というデモ的な事例は多いが、「実務で毎日使い続けている仕組み」の話は少ない——という問題意識がスタート地点。毎朝7時にジョブが走り、Slackを開くころには必要な情報が揃っている状態を目指した。

具体的な内容は記事本編に譲るが、CPOという非エンジニアが自分の業務に合わせた自動化を自作できているという事実自体が示唆に富む。営業やマーケティング担当者が自分専用の自動化を持つ時代が来ている。

## Agent Teams、「チームで開発して」と言うだけで並列作業が始まる

gihyo.jpでClaude Code Agent Teamsの衝撃が語られている。Agent Teamsは複数のAIエージェントが同時にタスクに取り組む機能で、プロンプトに「チームで行ってください」と指示するだけで発動する。

メインのエージェントが「チームリード」となり、専門チームメイトを生成。共有タスクリストが `~/.claude/tasks/` に作られ、チームメイトは自分でタスクを選んで「進捗どう？」「ここ終わった」と双方向で会話しながら作業を進める。従来のエージェントA→エージェントBという直列型ではなく、真の並列協調が起きる。

大規模リファクタリングではフロント・バックが同時に非推奨ライブラリを置き換え、数日かかる作業が数時間に。コードレビューでは複数エージェントがセキュリティ・パフォーマンス・設計など異なる観点から同時にチェックする。大規模な広告キャンペーンで媒体別・クリエイティブ別に並列レビューを走らせるイメージ。

[gihyo.jp](https://gihyo.jp/article/2026/02/get-started-claude-code-07)

## Teresa Torres、「/today」で毎朝のタスクリストを自動生成

プロダクトマネジメントの著名コンサルタントTeresa Torresが、Claude Codeを使った日常管理システムを公開している。2つのターミナルとObsidianを組み合わせた構成だ。

毎朝「/today」と打つだけで、その日のやることリストが自動生成される。これが機能するのは、3層のコンテキストファイルを整備しているから。グローバル層には「常に実行前に計画する」などの普遍的なルール、プロジェクト層にはフォルダ別のスタイル指定、リファレンス層には事業詳細を置く。

Claude Codeは必要に応じてこれらを参照し、「自分のビジネスを理解したアシスタント」として動く。ブログ執筆のSEOキーワード調査なども委任している。YOMIKOのクライアント別コンテキストファイル整備にも同じ発想が使える。

[Creator Economy](https://creatoreconomy.so/p/automate-your-life-with-claude-code-teresa-torres)

## 今日の所感

共通しているのは「最初から完璧を目指さない」姿勢。Polymarketの取引システムも転職自動化も、小さく始めて対話の中で育てている。Claude Codeは「一発で完成品を出す魔法」ではなく、「一緒に育てる相棒」として使うのが正解なのだろう。
