# Claude Code 今日の発見 — 2026-04-02

「速くなった気がする」と「本当に速い」は別物。今日集まった事例は、AIツールの導入が単純な効率化ではなく、人間との協働設計そのものだと教えてくれる。

---

## 予測市場で+9.6%リターン、全部Claude Codeとの会話で作った

スウェーデンの開発者Örvar Karlssonが、Polymarketという予測市場プラットフォームで動く自動取引システムを、Claude Codeとの単一セッションで構築した。Polymarketは「次の大統領は誰か」「ビットコインは年末いくらか」といった未来の出来事に賭けられる分散型マーケットで、公式CLIがあるものの「毎回ゼロから始まる」という制約があった。

Karlssonはまず価格監視ループを作り、次にアービトラージスキャナーを追加。初回スキャンで182件の裁定機会を発見した。$20/取引・$50/日の上限を設けたガードレール付きで実際に取引を回し、8取引中6勝、+9.6%のROIを達成。特にNegRiskアービトラージ（複数候補の「YES」価格合計が$1未満になる歪み）は100%勝率だった。面白いのは、学習ループとして戦略スコアカードや教訓抽出まで自動化し、Claude Codeの永続メモリに保存している点。次のセッションでは「前回の反省」を踏まえた提案が出てくる仕組みだ。

広告運用のA/Bテスト結果から次の施策を自動提案するワークフローに、同じアーキテクチャが使えそう。

## 「20%速くなった」と感じて、実際は19%遅くなっていた

2025年のMETR（AI評価研究機関）の調査が、AIコーディングツールの不都合な真実を明らかにした。開発者たちは「作業が20%速くなった」と体感していたのに、客観的な計測では19%遅くなっていた。認知負荷が下がって時間を短く感じる一方、AIの出力をレビュー・修正する時間が膨らんでいたのだ。

Qiitaの記事がこの「効率の幻想」を掘り下げている。AI生成コードのデバッグに余計な時間がかかると答えた開発者は45%。修正ループの比率は35.8%に達する。さらに深刻なのは品質面で、AI生成コードの45%にセキュリティ脆弱性が含まれ、コード重複は8倍に増加したという報告もある。

記事が提案する処方箋は「協働のアーキテクチャ」。定型業務（テスト生成、ドキュメント作成）はAIに、文脈依存的な判断（アーキテクチャ設計、セキュリティ検証）は人間に。Rakutenが1,250万行のコードから活性化ベクトルを抽出した事例では、7時間で99.9%の精度を達成したが、「人間が随時方向性を示した」ことが鍵だったとエンジニアは語っている。クライアントへのAI導入提案で「完全自動化」を売りにするのは危険かもしれない。

## 858スター/24時間、Oh-My-ClaudeCodeが示すマルチエージェントの現在地

韓国の開発者Yeachan Heoが公開した「Oh-My-ClaudeCode」が、GitHub史上最速クラスの勢いでスターを集めている。Claude Codeの上に被せるオーケストレーション層で、32種類の専門エージェントと5つの実行モードを提供する。

注目すべきは自動モデルルーティング機能。単純なタスクはClaude Haikuに、複雑な推論はClaude Opusに自動で振り分けることで、30-50%のトークン削減を実現しながら品質を維持する。実行モードには、企画から検証まで段階的に進む「Team Mode」、大規模リファクタ向けの並列実行「Ultrawork」、タスクが100%検証されるまで自己修復を繰り返す「Ralph」などがある。

3-5倍の高速化という数字は魅力的だが、先述の「効率の幻想」と合わせて読むと、何をもって「速い」とするかの定義が重要になる。

[GitHub - Yeachan-Heo/oh-my-claudecode](https://github.com/yeachan-heo/oh-my-claudecode)

## 広告分析が3-5時間から「体感7分」に

Rimo社でレベニューオペレーション責任者を務める佐伯さんが、営業・マーケ部門でのClaude Code活用を公開している。BigQueryから広告データを抽出し、状況要約から改善提案まで自動生成する「スキル」を自作。Google広告の改善観点を70-100項目のチェックリストに体系化し、それをClaude Codeに読ませている。

最も劇的な改善は広告分析で、従来3-5時間かかっていた業務が「体感7分」に短縮された。Salesforce CLIも統合し、担当者別成績や月次予実を対話的に取得できる仕組みも構築済み。ただし「提案はAIに、最終判断は人間に」という原則を守り、自動実行による暴走リスクを回避している。

[Rimo - 営業・マーケでもここまでできる。Claude Codeで業務を"ほぼ"自動化した話](https://rimo.app/@rimo/general/claude_code_ad_reporting)

## 毎朝7時に勝手に動く、CPOが作った11個の自動化

アソビュー株式会社のCPO横峯樹氏が、「毎日使い続けている」AI自動化の実態をnoteで公開した。「デモ映えする一発芸と、業務に定着する仕組みは別物」という問題意識から、地味だが毎日使う自動化を11個構築。毎朝7時になると勝手に動き出し、必要な情報が揃っているという仕組みだ。

非エンジニアのCPOが、SlackとClaude Codeを組み合わせて実務レベルの自動化を回している点が興味深い。派手さはないが、「定着する仕組み」という視点は、クライアントへのAI導入支援で見落としがちなポイントかもしれない。

## 50万行のソースコード流出、しかし収穫もあった

3月31日、AnthropicがClaude Codeのソースコード約50万行を誤って流出させた。npmパッケージの更新時に、デバッグ用のマップファイルが混入したのが原因。セキュリティ研究者がX（旧Twitter）で指摘し、投稿は2,880万回以上閲覧された。流出コードのGitHubミラーは8.4万スターを超えている。

皮肉なことに、この流出からClaude Codeの内部アーキテクチャが明らかになった。特に注目されたのは「自己修復メモリ」の仕組み。固定されたコンテキストウィンドウの制約を克服するため、重要な情報を永続化し、必要に応じて復元するアーキテクチャが実装されていた。

Anthropicは「顧客データや認証情報は含まれていない」「人的ミスによるパッケージング問題であり、セキュリティ侵害ではない」と声明を出した。ただし、流出時間帯（3月31日 00:21-03:29 UTC）にnpmからインストールした場合、トロイの木馬が混入したバージョンを取得した可能性があり、ダウングレードと認証情報のローテーションが推奨されている。

[The Hacker News - Claude Code Source Leaked via npm Packaging Error](https://thehackernews.com/2026/04/claude-code-tleaked-via-npm-packaging.html)

## 5ステップを1クリックに、小さな自動化の積み重ね

中国のエンジニアがBilibiliで公開した動画が、Claude Codeの「正しい使い方」を示している。飛書（Feishu、ByteDanceが提供する企業向けコラボレーションツール）でドキュメントを作成するには5ステップ必要だった。この面倒さに耐えかねて、Claude Codeでスクリプトを書き、ワンクリック操作に短縮した。

動画の説明文にある「小さな自動化改善の積み重ねが、ワークフロー効率を大幅に向上させる」という一文が印象的。派手なマルチエージェントより、こういう地味な改善の方が定着しやすいのかもしれない。
